# Agents 基礎: 特化した自動ワークフロー

## 🤖 Agents とは？

**Agents** は、特定のレビュー・リファクタリング・設計タスクに特化した自律 Copilot ワークフローです。

人間のチームメンバー（例：シニア建築家・テストエキスパート）のように、独立して機能し、専門分野に深く焦点を当てられます。

---

## 🎯 Skills vs Agents

| 特性 | Skills | Agents |
|------|--------|--------|
| **形式** | パッシブ知識 | アクティブワーカー |
| **実行** | 参照型（読まれる） | 自律実行（コード読み込み・提案） |
| **ツール** | なし | ツールセット（ファイル読み込み・grep など） |
| **出力** | パターン情報 | 具体的な改善提案・レポート |
| **使用例** | 「メモリパターンを学ぶ」| 「このモジュールを const 正確性でレビュー」|

```yaml
# Qt Architecture Reviewer

## 役割

このエージェントは以下をチェック:
- シグナル/スロット接続パターン（PMF vs 文字列）
- メモリリークの潜在予兆
- UI レスポンシブネス問題
- 親-子所有権

## レビュー手順

1. `.cpp` と `.h` ファイルを読む
2. 接続パターンを grep で検索
3. メモリリーク予兆を分析
4. 改善提案をリスト化
```

## 🎓 このプロジェクトの Agents

### 1. **qt-architecture-reviewer**

対象: シグナル/スロット設計・メモリ安全性

## 💡 Agents を使う方法

### 方法 1: 明示的呼び出し
```
あなた: "qt-architecture-reviewer でこのシグナル/スロット設計をレビューしてください"

Copilot: [エージェント実行]
- ファイル読み込み
- 分析
- 改善提案をリスト化
```

### 方法 2: コンテキスト検出
複雑なレビューが必要な場合、Copilot が自動的に適切なエージェントを提案
