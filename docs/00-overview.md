# GitHub Copilot カスタマイズ: 完全ガイド

## 🎯 Copilot カスタマイズとは？

GitHub Copilot カスタマイズは、Copilot の動作を以下の要素を通じて形作ることができます：

1. **Rules（ルール）** — ワークスペース・ファイルレベルのコーディングガイドライン
2. **Skills（スキル）** — パッケージ化されたドメイン知識と設計パターン
3. **Agents（エージェント）** — 特化した自律ワークフロー
4. **Hooks（フック）** — アクション前後の自動化品質ゲート
5. **Project CLAUDE.md** — 統合されたプロジェクトコンテキスト

各要素は開発者の意図の異なるレベルをターゲットにしており、高レベルのガイダンスから原子的な自動化まで対応します。

---

## 📊 5 つのカスタマイズ要素

### 1️⃣ **Rules** (.instructions.md)

**何か**: Copilot のコード提案をガイドするテキストファイル

**スコープ**:
- ワークスペース級: `copilot-instructions.md`（すべてのファイル）
- ファイル級: `.instructions.md`（`applyTo` で特定パターン指定）

**例**:
- 「すべての C++ コードは const 正確性を使用する」
- 「Qt シグナル/スロット接続は PMF シンタックスを使用する」
- 「ファイルが 800 行を超える場合はモジュールに分割する」

**効果**:
- Copilot は提案をこのルールへの適合性を考慮する
- 強制されない（違反提案もあるが優先度が低い）

**対象**: 初心者向け / Phase 2

---

### 2️⃣ **Skills** (`.github/skills/*/SKILL.md`)

**何か**: 特定のドメインやパターンに関する構造化知識パッケージ

**スコープ**:
- プロジェクト特化スキル（`.github/skills/`）
- ユーザー級スキル（`~/.claude/skills/`）
- プロジェクト間で再利用可能

**フォーマット**:
- YAML frontmatter（メタデータ、使用法、コンテキスト）
- Markdown セクション（理論 + コンクリートな実装例）
- 実践的パターン、一般的なアドバイスではない

**例**:
- `qt-signal-slot-design/SKILL.md` — 効率的なシグナル/スロット接続設計方法
- `cpp-memory-patterns/SKILL.md` — スマートポインタ、RAII、所有権
- `error-handling/SKILL.md` — C++ 例外 vs エラーコード

**効果**:
- パターンマッチ時に Copilot が関連スキルを参照
- Rules より権威的です（リファレンス素材的）

**対象**: 中級者向け / Phase 3

---

### 3️⃣ **Agents** (`.github/agents/*.agent.md`)

**何か**: 特定のレビュー・リファクタリング・設計タスクに特化した自律 Copilot ワークフロー

**スコープ**:
- プロジェクト当たりカスタムエージェント
- スラッシュコマンドまたは明示的呼び出しで実行可能

**例**:
- `qt-architecture-reviewer.agent.md` — シグナル/スロット と メモリリーク監査
- `cpp-refactorer.agent.md` — const 正確性・パターン改善提案
- `test-generator.agent.md` — 実装からユニットテストスイート生成

**効果**:
- チームのスペシャリストメンバーのように機能
- ドメインに深く焦点を当てられる
- 自律または対話的に実行可能

**対象**: 上級者向け / Phase 4

---

### 4️⃣ **Hooks** (`hooks.json`, Pre/PostToolUse)

**何か**: ツール呼び出し前後に発動する自動アクション

**スコープ**:
- グローバル: `~/.claude/settings.json`
- プロジェクト固有: `.github/hooks.json` or `.claude/hooks/`

**例**:
- **PreToolUse**: CMakeLists.txt 行数検証（編集前）
- **PostToolUse**: clang-format 自動フォーマット + 静的解析（cppcheck）実行
- **Stop**: 最終ビルド検証

**効果**:
- 品質ゲートを自動的に強制（手動ステップ忘却なし）
- 繰り返しチェック時間を削減（フォーマット・リント）
- エラーを早期に検出

**対象**: 上級者向け / Phase 5

---

### 5️⃣ **Project CLAUDE.md** (`.claude/CLAUDE.md`)

**何か**: Copilot にプロジェクト全体を「ブリーフ」する単一統合ファイル

**スコープ**:
- プロジェクトルート（`.claude/CLAUDE.md`）
- セッション当たり 1 回読込

**内容**:
- プロジェクト構造・ビルドコマンド
- アーキテクチャダイアグラム（ASCII または Mermaid）
- 一般的なタスク・レシピ
- トラブルシューティング FAQ
- キー依存関係・バージョン

**効果**:
- Rules/Skills だけでは提供できない深いコンテキスト
- Copilot のコードベース理解を高速化
- Copilot からの明確化質問を削減

**対象**: すべてのレベル / Phase 6

## 🔀 相互作用方法

```
┌──────────────────────────────────────┐
│   開発者からのリクエスト              │
└────────────┬─────────────────────────┘
             │
    ┌────────┴──────────┐
    │                   │
 (シンプル)         (複雑・ドメイン)
    │                   │
    ▼                   ▼
  Rules             Project CLAUDE.md + Skills
  ├─ 基本的なコーディングガイド      (コンテキストを知る)
  ├─ ファイルレベル制約             (専門知識)
    │
    └──────┬──────────┬──────────┐
           │          │          │
           ▼          ▼          ▼
        コード生成  パターン提案  スタイル/アプローチ決定
         │
         └──────┬──────────┬──────────┐
                │          │          │
           ┌────▼────┐  ┌──▼────┐  ┌─▼──────┐
           │  Hooks  │  │ Agents │  │ Skills │
           │(整形)   │  │(審査)  │  │(参照)  │
           └─────────┘  └────────┘  └────────┘
                │          │          │
                └────────┬─┴──────────┘
                         ▼
            高品質・準拠したコード
```

## 🎓 各要素を使うべき時

| シナリオ | これを使う |
|--------|----------|
| 「常に 4 スペースでインデント」| Rules |
| 「Qt シグナル/スロット PMF シンタックス使用」| Rules |
| 「スマートポインタ所有権パターン」| Skills |
| 「アーキテクチャの完全なコードレビュー」| Agents |
| 「編集ごとに自動フォーマット」| Hooks |
| 「ビルド手順・構造・トラブルシューティング」| CLAUDE.md |
| 「複数パターン + コンテキスト一緒」| Skills + CLAUDE.md |
| 「品質強制 + 早期エラー検出」| Hooks |
| 「モジュールの自律リファクタリング」| Agents |

---

## 📈 対象別進行順序

```
初心者
├─ 開始: Rules (Phase 2)        [基本ガイダンス理解]
│  
中級者
├─ 追加: Skills (Phase 3)       [ドメインパターン学習]
├─ 追加: CLAUDE.md (Phase 6)    [深いコンテキスト提供]
│
上級者
├─ 追加: Agents (Phase 4)       [特化ワークフロー]
├─ 追加: Hooks (Phase 5)        [自動化ゲート]
└─ 組み合わせ: Agents + Hooks + Skills [複雑なパイプライン]
```
